# 留存

> 留存衡量用户是否在产品自然使用周期内持续获得价值，并区分连续留存、召回、扩张与流失行为。

**[English](README.md)｜[简体中文](README.zh.md)**

## 核心问题

- 什么重复价值让用户回来？
- 哪个 Cohort 周期符合产品频率？
- 召回与扩张如何同连续留存分开？

## 跨公司模式

有效模式通常把产品价值、目标行为、衡量周期和经济或质量护栏连接起来。不同公司使用的表面策略不同，但都需要验证早期信号能否延伸到持续价值。

## 公司证据

| 公司或人物 | 证据与方法 |
| --- | --- |
| Duolingo | 学习习惯、Streak、恢复、通知与召回。 |
| Facebook / Meta | 核心社交价值、留存 Cohort、增长会计与数据质量。 |
| Airtable | 有用工作流与下游多人协作。 |

## 关键指标

- Cohort 留存曲线
- 新增、留存、召回与流失用户
- 价值事件频率与间隔
- 扩张深度或协作者活动
- 通知关闭与质量护栏

## 可以迁移的方法

1. 定义重复价值事件与自然间隔。
2. 按起始时期、分群、来源和产品状态建立 Cohort。
3. 把召回与连续留存分开报告。
4. 研究持续行为在哪里变化，而不只优化提醒。

## 不应该直接复制的部分

- 不要把汇总 DAU 或回访用户当作留存曲线。
- 被计数行为缺少产品价值时，不要添加 Streak 或通知。

## 证据与局限

- 公司证据主要来自参与者访谈和公开材料，不是完整内部因果评估。
- 公司规模、市场、产品频率、监管与数据能力限制直接迁移。
- 发布方公布的增长与收入数字不用于证明单项方法有效。
