# Retention

> Retention measures whether users continue receiving value over the product's natural usage cycle. It distinguishes continuously retained, resurrected, expanded, and churned behavior.

**[English](README.md)｜[简体中文](README.zh.md)**

## Core Questions

- What recurring value causes return?
- Which cohort window matches the product frequency?
- How should resurrection and expansion be separated from continuous retention?

## Cross-Company Patterns

Effective systems connect product value, target behavior, measurement horizon, and economic or quality guardrails. Surface tactics differ, but early signals still need validation against durable value.

## Key Metrics

- Cohort retention curve
- New, retained, resurrected, and churned users
- Frequency and interval between value events
- Expansion depth or collaborator activity
- Notification opt-out and quality guardrails

## Transferable Method

1. Define the recurring value event and natural interval.
2. Build cohorts by start period, segment, source, and product state.
3. Report resurrection separately from continuous retention.
4. Investigate where retained behavior changes rather than optimizing reminders alone.

## What Not to Copy Directly

- Do not use aggregate DAU or returning users as a retention curve.
- Do not add streaks or notifications when the counted behavior lacks product value.
