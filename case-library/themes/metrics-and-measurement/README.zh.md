# 指标与衡量

> 指标连接已交付客户价值、可观察行为与商业经济性。有效系统会区分结果、驱动、护栏和诊断维度，而不是寻找一个完美数字。

**[English](README.md)｜[简体中文](README.zh.md)**

## 核心问题

- 哪个指标代表已交付价值？
- 哪项先行行为可以预测结果？
- 哪些护栏能防止局部优化破坏质量或经济性？

## 跨公司模式

横向能力只有在连接具体增长环节和客户价值时才有意义。成熟做法会同时设计流程、责任、数据、反馈与风险边界。

## 公司证据

| 公司 | 证据与方法 |
| --- | --- |
| Ramp | 回本周期、贡献毛利、获客与扩张，以及多个价值指标。 |
| Facebook / Meta | 留存、增长会计、Magic Moment 与数据债务治理。 |
| Airtable | 激活定义、第四周协作与实验护栏。 |
| 字节跳动 | 预测 LTV、多周期 ROI、归因与产品矩阵增量。 |

## 关键指标

- 基于价值的北极星指标
- Cohort 激活与留存
- 新增、留存、召回与流失用户
- 贡献毛利与回本周期
- 增量与置信区间

## 可以迁移的方法

1. 选择指标前先写清客户价值事件。
2. 区分结果、输入、护栏与诊断指标。
3. 为每个指标定义分母、周期、分群、负责人和决策。
4. 审计埋点，并用实际 Cohort 校准模型预测。

## 不应该直接复制的部分

- 不要把收入、DAU 或注册当作通用北极星。
- 不要用长期预测掩盖已经观察到的弱行为。

## 证据与局限

- 证据主要来自参与者访谈和公开材料，不是完整内部因果评估。
- 公司规模、品类、监管、数据与人才条件限制直接迁移。
- 发布方增长、收入、估值与客户数主张不用于证明单项方法。
