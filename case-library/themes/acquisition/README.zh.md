# 获客

> 获客是触达可能持续获得产品价值的人或账户的系统，包括自然、付费、产品驱动、合作伙伴与产品矩阵分发，但不把原始流量当作成功。

**[English](README.md)｜[简体中文](README.zh.md)**

## 核心问题

- 哪类受众具有产品可以解决的问题？
- 哪个渠道能用可信证据触达这类受众？
- 停止投放或促销后，获客行为是否仍然持续？

## 跨公司模式

有效模式通常把产品价值、目标行为、衡量周期和经济或质量护栏连接起来。不同公司使用的表面策略不同，但都需要验证早期信号能否延伸到持续价值。

## 公司证据

| 公司或人物 | 证据与方法 |
| --- | --- |
| 字节跳动 | 长周期 LTV、自动投放、素材系统与本地化。 |
| Facebook / Meta | 校园扩张顺序、国际化与移除扩张障碍。 |
| Lovable | 付费扩张前的信任、员工参与、发布与自然分发。 |
| Ramp | B2B 品牌、产品差异化、活动与增长工程。 |

## 关键指标

- 合格触达与访问率
- 按来源和 Cohort 划分的激活
- 增量 CAC 与回本周期
- 按来源划分的留存用户或账户
- 自然、直接、推荐与品牌需求

## 可以迁移的方法

1. 选择渠道前先定义受众与持续价值。
2. 区分增量获客和本来就会到来的用户。
3. 把渠道指标连接到激活、留存与经济性。
4. 只有质量与回本仍可信时才扩大。

## 不应该直接复制的部分

- 不要把曝光、点击、安装或注册等同于增长。
- 没有相同能力时，不要复制字节规模的自动化或 Lovable 的社交分发。

## 证据与局限

- 公司证据主要来自参与者访谈和公开材料，不是完整内部因果评估。
- 公司规模、市场、产品频率、监管与数据能力限制直接迁移。
- 发布方公布的增长与收入数字不用于证明单项方法有效。
