# 激活

> 激活是新用户已经获得有意义产品价值的第一项可靠证据。它是需要通过后续价值验证的行为假设，而不是完成注册或新手引导。

**[English](README.md)｜[简体中文](README.zh.md)**

## 定义与边界

激活回答一个具体问题：**新用户是否已经获得足够的价值，从而更有可能继续使用产品？** 回答这个问题通常需要一项可观察的价值事件、明确的时间窗口，以及该事件与后续留存或其他持续结果相关的证据。

激活与下列概念相关，但不能直接画等号：

| 概念 | 衡量对象 | 为什么不能单独代表激活 |
| --- | --- | --- |
| 注册 | 创建账户或进入产品 | 获得访问权限不等于已经获得价值。 |
| 完成新手引导 | 走完产品设计的流程 | 用户可能完成界面操作，却没有得到有用结果。 |
| 参与度 | 活动频率或数量 | 活动可能很浅、偶然发生，或者与价值无关。 |
| 留存 | 一段时间内持续获得价值 | 留存能够验证激活，但发生在更靠后的阶段。 |

不存在通用的激活事件。协作产品可能既要产出有用成果，也要有其他参与者；消费工具可能在完成一次重要任务时激活；双边市场还需要同时考虑供需两侧的价值。

## 激活模型

与其把激活理解成一个二元事件，不如把它拆成四个相互连接的阶段。

| 阶段 | 核心问题 | 证据 | 常见失败 |
| --- | --- | --- | --- |
| **1. 用户意图** | 用户为什么来到这里？ | 进入来源、声明的用途、角色、导入的上下文 | 不同意图的用户被送入同一条通用路径。 |
| **2. 首次价值** | 用户是否完成了一项有意义的任务？ | 成功发生的价值事件，而不是界面事件 | 流程优化的是设置步骤，不是用户结果。 |
| **3. 价值强化** | 用户是否加深、重复或分享了价值？ | 第二次使用、更丰富的成果、协作、保存状态 | 第一次成功是孤立事件，没有创造回访理由。 |
| **4. 留存验证** | 候选激活事件能否预测持续价值？ | Cohort 留存、扩展或重复完成任务 | 一个方便统计的早期事件被误认为因果节点。 |

这个模型并不要求产品必须设计四个独立界面或事件。它要求衡量系统能够区分获客意图、首次成功、价值强化和下游验证。

## 指标树

单一激活率无法解释用户为什么成功或失败。指标树中的每一层需要回答不同的决策问题。

```text
持续客户价值
└── 留存激活
    ├── 激活结果
    │   ├── 激活率
    │   └── 首次价值时间
    ├── 过程诊断
    │   ├── 步骤转化率
    │   ├── 放弃节点
    │   └── 步骤间耗时
    ├── 激活质量
    │   ├── 价值深度 / 广度
    │   ├── 重复价值
    │   └── 协作或后续使用
    └── 护栏
        ├── 错误与任务失败
        ├── 支持成本
        └── 留存、信任与经济性影响
```

- **结果指标**说明用户是否到达候选价值节点。
- **过程指标**定位摩擦，但不能证明已经交付价值。
- **质量指标**区分表面完成与有意义的成功。
- **护栏指标**防止局部转化提升损害信任、产品质量、留存或经济性。

## 指标字典

发布任何激活指标之前，都要定义合格用户、起算事件、价值事件、观察窗口、排除条件与负责人。改变其中任何一项，得到的都是另一个指标。

| 指标 | 定义与公式 | 窗口与有效拆分 | 解读方式 |
| --- | --- | --- | --- |
| **激活率** | `W 时间内完成价值事件的用户数 / Cohort 中符合条件的新用户数` | 首次会话、24 小时、7 天；来源、意图、用途、角色、设备、市场 | 核心结果指标。分母不能悄悄排除遭遇摩擦的用户。 |
| **首次价值时间（TTFV）** | `首次价值事件时间 - 激活起算时间`；同时报告中位数与分位数 | 与激活率一致的 Cohort 周期；意图、来源、设备、产品复杂度 | 只有事件代表真实价值时，更快才有意义；中位数会隐藏长尾。 |
| **步骤转化率** | `完成步骤 n 的用户数 / 符合步骤 n 条件的用户数` | 按会话与 Cohort；路径、设备、实验版本 | 诊断指标。流程完成率很高，激活仍可能很弱。 |
| **步骤放弃率** | `到达步骤 n 后未进入 n+1 的用户数 / 到达步骤 n 的用户数` | 会话超时加明确回访窗口；路径与错误状态 | 说明进度在哪里停止，不能单独说明原因；需要结合错误与定性证据。 |
| **留存激活率** | `在 R 周期重复价值事件的激活用户数 / 符合回访条件的激活用户数` | 按产品自然频率选择日、周或月；激活路径与 Cohort | 质量结果指标。回访周期必须符合自然使用周期。 |
| **激活带来的留存差值** | `激活用户留存率 - 可比未激活用户留存率` | 固定留存周期；在来源、意图和 Cohort 内比较 | 验证预测价值，不等于因果关系；选择效应需要实验或更强的因果设计。 |
| **价值深度 / 广度** | 首次成功后获得价值的程度或范围，例如有用对象、完成任务、服务同一任务的功能或协作者数量 | 早期窗口加下游 Cohort；用途与账户类型 | 质量诊断指标。更多活动不一定代表更多价值。 |
| **错误率** | `相关任务的失败尝试次数 / 总尝试次数` | 按步骤、平台、集成、地区与版本 | 护栏与诊断指标。必须说明重试是否算作新的尝试。 |
| **支持成本** | `激活相关咨询或人工协助次数 / 符合条件的新用户数` | Cohort 与发布周期；问题类型与用户分群 | 依赖大量人工支持的转化提升可能无法规模化。 |

### 计算示例

假设一周内有 1,000 个符合条件的新工作区开始使用产品，其中 420 个在 7 天内创建了有用项目并完成第一个目标工作流：

```text
7 日激活率 = 420 / 1,000 = 42%
```

如果这 420 个工作区中有 210 个在第四周再次完成该工作流：

```text
第 4 周留存激活率 = 210 / 420 = 50%
```

这些数字不是行业基准。只有在相同定义下比较不同 Cohort 和分群，并确认价值事件能够预测持续客户结果时，它们才有意义。

## 分群与 Cohort 分析

整体激活率变化，经常是因为用户构成发生变化，而不是产品真正改善。分析应从获客 Cohort 开始，并只选择能够改变解释或行动的维度：

| 维度 | 帮助回答的问题 |
| --- | --- |
| 来源或 Campaign | 用户进入产品之前形成的预期是否与实际体验一致？ |
| 意图或用途 | 不同任务的用户是否被迫走同一条路径？ |
| 角色或账户类型 | 执行激活的人是否具备获得价值所需的权限、数据或协作者？ |
| 设备、平台或版本 | 摩擦来自体验设计还是技术可靠性？ |
| 市场、语言或地区 | 内容、集成和价值假设能否迁移？ |
| 新用户、受邀用户或回访用户 | 起始上下文明显不同的用户是否被混在一起？ |

比较 Cohort 时必须使用相同的合格条件与成熟窗口。不能把已经完整观察 7 天的 Cohort 与昨天刚进入的用户直接比较。同时报告绝对人数和比率，避免规模很小但比例很高的分群主导结论。

## 激活问题诊断

观察到的模式只能缩小调查范围，不能直接证明根因。每个模式都应导向明确的检查与下一步行动。

| 观察到的模式 | 可能的问题 | 下一步检查 | 可能的行动 |
| --- | --- | --- | --- |
| 注册很多，第一步启动率很低 | 预期错位、入口不清楚、缺少前置条件 | 按来源与意图看 Cohort、入口行为、短访谈 | 对齐获客承诺、预填上下文或按意图分流。 |
| 引导完成率高，价值事件完成率低 | 流程奖励界面完成，而不是用户任务 | 比较引导步骤与留存用户的真实成功路径 | 删除非必要设置，把产生价值的操作提前。 |
| 首次价值不错，但 TTFV 很慢 | 依赖、导入、权限或决策太多 | 按步骤、角色、平台与错误查看 TTFV 分位数 | 使用模板、渐进设置、默认值或异步准备。 |
| 激活率高，留存激活率低 | 激活定义太弱、成功质量低、没有重复价值 | 按激活深度看留存、访谈、重复任务行为 | 提高或重定义价值阈值，并设计价值强化。 |
| 整体表现强，关键分群很弱 | 用户结构变化，或路径只服务主要用途 | 按来源、意图、角色、市场和设备拆分 | 只有需求和产品行为确实不同时才建立独立路径。 |
| 转化提升，但错误或咨询增加 | 摩擦被隐藏或转移到下游 | 错误分类、支持咨询、取消与信任信号 | 回滚、限制覆盖范围，或在护栏约束下重做。 |

## 实验与验证方法

1. **写明价值假设。** 描述客户任务，以及能够表示有意义完成的行为。
2. **固定衡量契约。** 在读取结果前记录合格条件、分子、分母、起算事件、窗口、排除条件、分群和护栏。
3. **验证预测关系。** 比较成熟 Cohort，确认候选事件是否先于更强留存、重复价值、扩展或其他持续结果。
4. **定位主要约束。** 结合过程指标和定性证据，找出价值路径在哪里、为什么中断。
5. **改变一个机制。** 根据诊断测试更清晰的承诺、更少的前置条件、模板、默认值、引导操作或强化机制。
6. **读取完整指标树。** 同时评估激活、质量、留存激活、错误、支持、信任和经济性护栏。
7. **持续重新验证。** 产品和获客结构会变化，曾经能预测留存的激活定义可能逐渐失效。

A/B 实验可以估计某项干预的影响，但不能挽救一个薄弱的价值定义。反过来，早期事件与留存之间很强的观察关系可以用于预测，却仍可能来自用户选择，而不是该事件本身的因果作用。

## 操作方法

为每个产品或具有实质差异的用途填写以下定义：

```text
用户与意图：
客户任务：
合格用户：
激活起算事件：
首次价值事件：
观察窗口：
价值强化信号：
留存验证周期：
主要分群：
质量指标：
护栏指标：
决策负责人：
现有证据与置信度：
```

当产品、目标客户、获客结构或自然使用频率发生变化时，需要重新审查这份定义。如果一套激活定义无法解释差异明显的客户任务，应保留多套定义，而不是把不兼容的路径平均在一起。

## 不应该直接复制的部分

- 不要复制其他产品的 Aha Moment、事件阈值或时间窗口。
- 没有价值证据时，不要把注册、完成向导、关闭 Tooltip 或发出邀请提升为激活事件。
- 不要只优化激活率，而忽略留存激活、错误、信任、支持成本或单位经济性。
- 没有处理选择效应与混杂因素时，不要把激活和留存之间的相关关系解释成因果证明。
- 不要为了产生更多图表而增加无法改变决策或产品路径的分群。

## 证据与局限

- 公司证据主要来自参与者访谈和公开材料，不是完整的内部因果评估。
- 激活事件和阈值是依赖产品、用户分群与时间的假设。
- 公司规模、市场、产品频率、监管与数据能力限制直接迁移。
- 发布方公布的增长、收入、估值、客户数与用户数不用于证明单项方法有效。
