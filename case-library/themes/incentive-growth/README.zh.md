# 激励增长

> 激励增长使用现金、金币、优惠券、恢复、身份或权限改变行为。只有激励强化产品价值，并在成本、作弊和蚕食后创造持续增量时，它才有效。

**[English](README.md)｜[简体中文](README.zh.md)**

## 核心问题

- 需要改变哪项行为，它为什么有价值？
- 奖励属于货币、功能、社交还是保护？
- 激励减少或停止后还剩下什么？

## 跨公司模式

横向能力只有在连接具体增长环节和客户价值时才有意义。成熟做法会同时设计流程、责任、数据、反馈与风险边界。

## 公司证据

| 公司 | 证据与方法 |
| --- | --- |
| 字节跳动 | 现金、金币、优惠券、裂变奖励、极速版、人群隔离与反作弊。 |
| Duolingo | Streak、Streak Freeze、里程碑、通知与付费恢复边界。 |
| Facebook / Meta | 围绕网络价值建设的邀请与连接机制。 |

## 关键指标

- 资格到行动提升
- 持续增量行为
- 奖励与作弊成本
- 蚕食与毛利
- 激励后留存

## 可以迁移的方法

1. 定义分群、行为、资格、奖励、周期与停止条件。
2. 使用对照组估计增量。
3. 区分奖励行为与自然产品信号。
4. 衡量产品矩阵蚕食、信任与作弊。

## 不应该直接复制的部分

- 没有风控能力时，不要上线现金系统。
- 不要把被奖励的时长或邀请当作满意度证据。

## 证据与局限

- 证据主要来自参与者访谈和公开材料，不是完整内部因果评估。
- 公司规模、品类、监管、数据与人才条件限制直接迁移。
- 发布方增长、收入、估值与客户数主张不用于证明单项方法。
