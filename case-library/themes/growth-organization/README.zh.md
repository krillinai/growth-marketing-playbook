# 增长组织

> 增长组织围绕增长约束定义责任、决策权、跨职能权限与学习系统。目标不是一种通用组织图，而是对客户价值的协调责任。

**[English](README.md)｜[简体中文](README.zh.md)**

## 核心问题

- 哪项约束需要专门负责人？
- 哪些职能必须共同改变产品、分发、数据或经济性？
- 什么语境与权限能让团队快速决策又不失控？

## 跨公司模式

横向能力只有在连接具体增长环节和客户价值时才有意义。成熟做法会同时设计流程、责任、数据、反馈与风险边界。

## 公司证据

| 公司 | 证据与方法 |
| --- | --- |
| Ramp | 聚焦授权、增长工程、实验组合、写作与互补人才。 |
| 字节跳动 | UG 共享平台、本地执行、自动化与风险隔离。 |
| Facebook / Meta | 跨职能增长团队、埋点、产品权限与共享留存目标。 |
| Lovable | 员工参与分发，以及建设与沟通的紧密连接。 |

## 关键指标

- 决策与学习周期
- 跨职能依赖等待
- 实验与发布跟进
- 按团队划分的客户价值结果
- 质量、事故与可持续性护栏

## 可以迁移的方法

1. 选择组织形式前先明确当前约束。
2. 给团队语境、产品权限、清晰结果与明确决策权。
3. 集中共享能力，同时保留市场与产品判断。
4. 用评审和复盘改善运行系统，而不是追责。

## 不应该直接复制的部分

- 不要招聘增长负责人来制造 PMF。
- 不要复制大公司平台团队，也不要要求每位员工都成为营销人员。

## 证据与局限

- 证据主要来自参与者访谈和公开材料，不是完整内部因果评估。
- 公司规模、品类、监管、数据与人才条件限制直接迁移。
- 发布方增长、收入、估值与客户数主张不用于证明单项方法。
